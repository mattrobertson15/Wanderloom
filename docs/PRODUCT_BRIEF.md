# Product Brief

## Positioning

Wanderloom is a globe-first social travel scrapbook and discovery platform. It sits between three categories that all currently solve the problem partially:

- **Photo journals / scrapbook apps** (private, beautiful, but not social or discoverable)
- **Generic map/travel-planning utilities** (functional, but cold and listicle-driven)
- **Social travel feeds** (social, but chronological-feed-first, not place/map-first and not built around durable trip structure)

Wanderloom's bet: the most trustworthy travel recommendation is a friend's real trip, rendered on a map you can explore. The product is built backwards from that insight — personal scrapbooking is the content engine, and the globe is the discovery surface.

## Target Users

1. **Frequent travelers / travel creators** — want a beautiful, structured home for their trips and the social reach of an audience.
2. **Casual travelers** — want an easy way to remember and organize trips with friends/family, without needing public reach.
3. **Trip planners / dreamers** — browse friends' and public travelers' maps to decide where to go next, save places, and build their own future trips from inspiration.

## Core User Loops

**Creation loop (scrapbook):**
```
Create Trip → Add Albums → Add Posts + Photos → Attach Places → Pins render on globe
```

**Discovery loop (social):**
```
Browse globe/map → View friend or public trip → Save place / follow user → Start own trip
```

**Sharing loop (growth):**
```
Finish trip → Set visibility (public/friends) → Share trip link → Recipient views → Signs up → Follows/friends
```

These loops are designed to reinforce each other: every act of personal scrapbooking produces content that feeds discovery, and every discovery session can convert into new scrapbook activity.

## Differentiation

- **Trip-first, not post-first.** Content is anchored to a Trip the user already cares about, not a generic feed post. This produces durable, structured, revisitable content instead of ephemeral posts.
- **Global, reusable Places.** Place data (e.g. "Statue of Liberty") is shared across all users and trips, enabling map density and cross-user discovery (roadmap: "see everyone's posts about this place").
- **Globe as the home surface**, not a utility bolted onto a feed.
- **Real trips as recommendations**, not aggregated/generic "Top 10" content.

## Primary Use Cases

- Document a trip you just took or are currently taking, with friends/family able to follow along.
- Build a public travel profile / portfolio as a creator.
- Browse a friend's past trips before visiting the same place.
- Discover new destinations via public travelers' maps.
- Share a single trip via a link with someone outside the app (friends, family, prospective followers).

## Growth Loops

- **Share-link virality**: public/friends trip links shared outside the app drive signups.
- **Follow-based discovery**: public profiles and trip pages are crawlable/shareable surfaces that convert visitors into followers.
- **Friend-invite loop**: friend requests pull in people who already travel with the user.
- **Map density loop**: more users → more Posts per Place → richer discovery → more reason to browse → more users.

## Non-Goals (MVP and near-term)

- Not a trip-booking or itinerary-execution tool (no flights/hotels/payments).
- Not a generic listicle/guide content site.
- Not a video-first platform (photos only for MVP).
- Not an open social feed/timeline product — the map/globe and Trip pages are the primary navigation surfaces, not an infinite scroll feed.
- Not ad-monetized at MVP (ads are a roadmap monetization direction, not a launch feature).
- Not building proprietary map tiles/geocoding — Mapbox is the rendering layer, not a strategic asset.
